# HAL 9000 Knowledge Base

A curated reference library the on-device model retrieves against at inference
time. Focus: ISS operations, the "big three" emergencies, and core spacecraft
systems.

Structure:

- `ammonia/` — primary demo scenario: detection, immediate actions, isolation,
  air scrubbing, crew health, when to abandon ship.
- `emergencies/` — overview of fire, rapid depressurization, toxic atmosphere,
  micrometeoroid strike, and the alarm taxonomy that drives the response.
- `systems/` — major ISS subsystems (ECLSS, EPS, thermal, comms, GNC) and the
  module/port layout.
- `operations/` — crew schedule, roles, health baselines, ground communication
  windows, consumables.

**Source & disclaimer.** Content synthesized from public NASA materials: the
ISS Reference Guide, NASA technical reports (NTRS), astronaut training
materials released for public education, and Wikipedia. This is **not** the
ISS Flight Data File. Specific emergency checklists, panel IDs, and cue card
verbiage in the real FDF are controlled and not reproduced here. These files
describe the *structure* and *reasoning* of ISS emergency response at a level
appropriate for a training aid, not for actual on-orbit execution.
