# HAL 9000 Voice

TTS backend for the HAL 9000 voice agent. Uses [Piper](https://github.com/OHF-Voice/piper1-gpl)
(the OHF-Voice GPL fork of Piper TTS) with a voice fine-tuned on
the 96-clip HAL dataset bundled in this directory.

## Layout

```
voice/
├── audio/              96 source clips, HAL 9000 lines from the film
├── metadata.csv        file_name,text — pairing clips with transcripts
├── hal9000_reference.wav  concatenated reference (legacy; unused by Piper)
├── preprocess.py       converts audio/ + metadata.csv to LJSpeech format
├── hal_tts.py          inference — loads hal.onnx, used by server/tts.py
├── requirements.txt    piper-tts, onnxruntime, soundfile (inference only)
├── ljspeech/           generated by preprocess.py (gitignored)
├── hal.onnx            trained model weights (gitignored; train to produce)
└── hal.onnx.json       voice config produced by the Piper exporter (gitignored)
```

## Inference (server uses this path)

Once `hal.onnx` and `hal.onnx.json` exist in this directory,
`server/tts.py` will pick them up automatically — nothing else
to wire. If they don't exist, the server falls back to macOS
`say`, so you can keep developing while training runs elsewhere.

Standalone sanity check:

```bash
python -c "import hal_tts; import soundfile as sf; \
  w, sr = hal_tts.synthesize('I am a HAL nine thousand computer.'); \
  sf.write('/tmp/hal.wav', w, sr)"
afplay /tmp/hal.wav
```

## Training

Piper training lives in a separate process with heavy deps
(PyTorch, phonemizer, espeak-ng). Easiest to do it on a GPU
machine (Colab free tier works) and copy the resulting ONNX
back here.

### 1. Prepare the dataset

On this machine (or wherever you run training):

```bash
cd voice
python preprocess.py
```

This converts the HuggingFace-style `metadata.csv` into
LJSpeech layout under `voice/ljspeech/` — that's what piper1-gpl's
training pipeline expects.

### 2. Install piper1-gpl in a training env

```bash
# Fresh Python 3.10/3.11 venv is cleanest.
git clone https://github.com/OHF-Voice/piper1-gpl piper-train
cd piper-train
python3.11 -m venv venv
source venv/bin/activate
pip install -e .[train]
```

### 3. Preprocess into Piper's internal cache format

```bash
python -m piper.train.preprocess \
    --language en_us \
    --input-dir ../voice/ljspeech \
    --output-dir ../voice/piper_cache \
    --dataset-format ljspeech \
    --single-speaker \
    --sample-rate 22050
```

### 4. Fine-tune from an English checkpoint

Start from a pretrained checkpoint (e.g., `en_US-lessac-medium`)
to avoid training from scratch — the 96-clip HAL set is too small
to learn a voice from zero.

```bash
# Download the base checkpoint once:
wget https://huggingface.co/rhasspy/piper-voices/resolve/v1.0.0/en/en_US/lessac/medium/en_US-lessac-medium.ckpt

python -m piper.train fit \
    --data.voice-name hal \
    --data.csv-path ../voice/ljspeech/metadata.csv \
    --data.audio-dir ../voice/ljspeech/wavs \
    --data.espeak-voice en-us \
    --data.cache-dir ../voice/piper_cache \
    --ckpt-path en_US-lessac-medium.ckpt \
    --trainer.max_epochs 2000 \
    --trainer.accelerator gpu \
    --trainer.devices 1 \
    --batch-size 32
```

Expect 1–3 hours on a T4 for the HAL dataset to sound close to
the reference. Watch the sample WAVs that Piper dumps during
training — stop when they sound right.

### 5. Export to ONNX and move back

```bash
python -m piper.train.export_onnx \
    --checkpoint lightning_logs/version_0/checkpoints/last.ckpt \
    --output-file hal.onnx

cp hal.onnx hal.onnx.json ../voice/
```

Restart the HAL server; next reply will be in HAL's voice.

## Notes

- 96 clips of ~10-15 min total is *small* for voice training. The
  reference (pretrained) checkpoint does most of the heavy lifting;
  the HAL fine-tune just shifts the voice. Don't expect a perfect
  Douglas Rain — expect a recognisable impression.
- If training output sounds robotic or glitchy, train longer or
  check your preprocessing (audio sample rate, normalisation).
- `voice/hal9000_reference.wav` is left in the repo because it's
  small and a useful reference cut if anyone wants to try a
  different voice-clone approach later (Qwen3-TTS, F5-TTS, etc.).
